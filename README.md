# FindOPedia

A web application where users discover random English Wikipedia articles and permanently claim ownership by passing an AI-generated comprehension quiz. The first user to score 100% becomes the sole owner of that article forever.

## How it works

- Click "Discover" to fetch a random Wikipedia article
- Read the article in full on the page
- Take a 10-question quiz generated by an AI model of your choice
- Score 100% to claim the article; anything less and you cannot retry
- One attempt per user per article, enforced at the database level
- Owned articles go to your collection; others can still read but never claim
- Articles are categorized by rarity based on their content length
- 4-hour cooldown between successful ownership claims

## Rarity tiers

| Tier | Content length |
|---|---|
| Common | under 5,000 bytes |
| Uncommon | 5,000 to 19,999 bytes |
| Rare | 20,000 to 49,999 bytes |
| Epic | 50,000 to 99,999 bytes |
| Legendary | 100,000 bytes and above |

## Stack

- **Backend**: Go, chi router, PostgreSQL, JWT auth (HS256), bcrypt passwords
- **Frontend**: Vite, React, TypeScript, Zustand, React Router
- **Deployment**: Docker Compose; only the frontend is exposed externally

## AI quiz generation

The quiz is generated by an AI provider of your choice. The API key is stored in your browser's localStorage only and is never sent to or stored on the server. It is transmitted directly in the quiz request and discarded afterward.

Supported providers:
- OpenAI (gpt-4o-mini)
- Google Gemini (gemini-2.0-flash)
- Anthropic Claude (claude-haiku-4-5)

Configure your provider and key on the Settings page. Quiz questions are cached in the database after the first generation, so subsequent users see the same questions without triggering another AI call.

## Running locally

Prerequisites: Docker, Docker Compose.

```bash
cp example.env .env
# Edit .env and set a strong JWT_SECRET (minimum 32 characters)

JWT_SECRET=$(openssl rand -hex 32) docker compose up --build
```

The application will be available at `http://localhost:3000`.

The backend and database are internal to Docker and not exposed on the host.

## Environment variables

| Variable | Required | Description |
|---|---|---|
| `JWT_SECRET` | Yes | Secret key for signing JWT tokens, minimum 32 characters |
| `APP_PORT` | No | Backend port, defaults to 8080 |
| `POSTGRES_DSN` | No | Override the default PostgreSQL connection string |
| `JWT_EXPIRY_HOURS` | No | Token lifetime in hours, defaults to 168 (7 days) |
